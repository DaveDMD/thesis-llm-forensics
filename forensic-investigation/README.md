# Forensic chain — timeline · attribution · IR playbook · custody

A **cross-cutting, model-agnostic** demonstration of the forensic capability: given the
hash-chained evidence log produced by the service, reconstruct what happened, attribute
sessions to actors, run the incident-response playbook, and preserve the evidence.

Unlike the per-target campaigns (`campaigns/`), this is **not** a per-model result. The chain
operates on **request metadata and log structure, not on model outputs** — staged against
gpt2, Pythia or Mistral it yields the same result. It is therefore demonstrated **once**, on a
realistic multi-actor incident. (Per campaign, only the no-GPU custody verification is run.)

## The scenario

A realistic incident with **distinct actors**, so attribution is discriminating:
- one adversary running **two extraction sessions** (same network identity),
- a **second adversary** running a divergence-extraction session,
- **benign** traffic from several distinct legitimate users.

## What it produces (committed run)

- **(1) Timeline** — 10 sessions reconstructed; phase `extraction_attempt` observed on the 2
  adversary sessions.
- **(2) Attribution** — **1** discriminating link: the adversary's two sessions are tied
  together (shared IP / ASN / user-agent hash + temporal proximity, confidence 0.90); benign
  sessions are not linked. Attribution is a ranked list of **prudential hypotheses**, not identity.
- **(3) IR playbook** — triage **severity: high** (2 suspicious sessions); snapshot of
  model+index+log components; preservation (WORM + chain verification); escalation:
  *"escalate to incident lead; preserve snapshot; consider GDPR breach assessment"*.
- **(4) Chain of custody** — independent hash-chain verification: **ok** (172 records).

## How to run

From the repository root (everything runs in the project Docker image):

```bash
# No-GPU replay (default): re-derive timeline structure + attribution + custody live from the
# committed stream, then show the committed full IR report (phases, triage, escalation).
forensic-investigation/reproduce.sh --replay

# Full GPU regeneration: re-stage the multi-actor incident against the model and rebuild it.
forensic-investigation/reproduce.sh --regen
```

The no-GPU replay reproduces the model-agnostic parts faithfully (timeline structure, the
discriminating attribution link, custody verification). Phase tagging and triage severity use
the labelled feature view and are shown from the committed IR report (regenerated by `--regen`).
